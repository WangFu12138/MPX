# 项目提案：基于视觉端到端可微MPC的双足机器人非结构化地形运动控制
**Project Proposal: Visual End-to-End Differentiable MPC for Bipedal Locomotion on Unstructured Terrain**

---

## 1. 背景与文献调研 (Background & Literature Review)

### 1.1 现状分析：当前双足机器人控制的三大主流范式
目前双足机器人（如Cassie, Digit, Unitree H1）的控制方案主要分为三类，各有明显痛点：

*   **范式一：经典分层MPC (Hierarchical MPC)**
    *   **代表作**：MIT Cheetah 3 MPC (Di Carlo et al., 2018), Convex MPC (Kim et al., 2024).
    *   **流程**：感知构建高程图 -> 规划落脚点 -> MPC 跟踪。
    *   **痛点**：**感知与控制解耦**。感知模块产生的地图误差（如把草丛当成石头）对于 MPC 来说是不可见的，MPC 只能盲目执行，导致在软地面或欺骗性视觉特征上表现极差。
*   **范式二：端到端强化学习 (End-to-End RL)**
    *   **代表作**：Learning to Walk in Minutes (ETH Zurich, 2023), PPO-based Locomotion.
    *   **流程**：神经网络直接从像素映射到关节位置/力矩。
    *   **痛点**：**样本效率低，不可解释，安全性差**。RL 需要数亿步采样才能学会走路，且是一个“黑盒”，一旦真机摔倒，很难分析是视觉错了还是动力学错了。
*   **范式三：视觉残差学习 (Visual Residual Learning)**
    *   **代表作**：RMA (Rapid Motor Adaptation), WildCat (2024).
    *   **流程**：RL 输出一个修正项加在 MPC 或 WBC 上。
    *   **痛点**：虽然提升了鲁棒性，但**核心控制器（MPC）内部的参数依然是固定的**，并没有利用视觉信息去改变 MPC 的优化目标本身。

### 1.2 现有“可微控制”研究及局限性
*   **Diff-MPC / AC-MPC (Actor-Critic MPC)**：主要用于无人机或简单小车。
    *   *局限*：大多未处理双足机器人复杂的**断续接触（Hybrid Contact Dynamics）**问题。
*   **Differentiable Physics (Brax, Dojo, MJX)**：解决了物理引擎可导的问题。
    *   *机会*：2024-2025年，随着 **JAX** 生态的成熟和 **MPX** (iLQR in JAX) 的出现，计算速度终于能满足 100Hz+ 的控制需求，为本项目的可行性奠定了基础。

## 2. 核心创新点 (Core Innovations)

旨在提出一种 **“视觉感知与动力学控制深度融合”** 的新架构，核心在于利用 **可微 MPC (Differentiable MPC)** 作为中间层，打通从像素到力矩的梯度流。

### 2.1 核心引擎 (The Engine)

1.  **MPX (Model Predictive Control in JAX)**
    *   **GitHub**: [`iit-DLSLab/mpx`](https://github.com/iit-DLSLab/mpx)
    *   **论文**: *Primal-Dual iLQR for GPU-Accelerated Learning and Control in Legged Robots* (RA-L/ICRA 2025, ArXiv 2412)
    *  它证明了在 GPU 上用 JAX 写 MPC 可以达到 100Hz+ 的频率，且支持自动微分。把它的 `cost_function` 替换成神经网络的输出。

2.  **MpcJax**
    *   **GitHub**: [`google-deepmind/mujoco_playground`](https://github.com/google-deepmind/mujoco_playground) (内含 MJX 控制示例)
    *   **相关库**: `hydrax` (by Vince Kurtz), `trajax` (Google Research)
    *   **重要**：Google DeepMind 官方的 JAX 仿真。里面直接就有 **Unitree H1 / G1** 的双足机器人模型。直接在这个 Playground 里把 RL 替换成 Diff-MPC 即可。

### 创新点 A：基于 SDF 的可微地形代价函数 (Differentiable Terrain Cost via SDF)
*   **传统做法**：将地形离散化为栅格地图（Occupancy Grid），不可导。
*   **本项目**：训练视觉网络输出**隐式符号距离场 (Implicit SDF)** 或 **参数化地形权重图**。
*   **优势**：SDF 是连续可导的。当 MPC 规划的轨迹穿过障碍物时，可以直接计算出 $\frac{\partial Cost}{\partial Trajectory}$，并进一步通过链式法则计算 $\frac{\partial Trajectory}{\partial VisionNet}$，从而指导视觉网络“修正”其对地形的理解。
3.  **iSDF: Real-Time Neural Signed Distance Fields for Robot Perception**
    *   **链接**: [Project Page](https://joeaortiz.github.io/iSDF/) | [Paper (RSS 2022)](https://arxiv.org/abs/2204.02296)
    *   **核心思想**：实时从 RGB-D 相机学习一个连续的 SDF 场。
    *  抄它的网络架构。把它的输出 $SDF(x)$ 变成 MPC 里的障碍物惩罚项 $C_{obs} = \text{ReLU}(r_{robot} - SDF(x))$。iSDF 是神经网络，所以全链路可导！
4.  **Differentiable Rendering / Physics (基础理论)**
    *   **论文**: *End-to-End Differentiable Physics for Learning and Control* (NeurIPS 2018)
    *   **链接**: [Paper](https://arxiv.org/abs/1806.09505)
    *   **祖师爷**级别的文章。虽然它是做刚体推箱子的，但它从数学上证明了“把物理引擎当成神经网络的一层”是可行的。
### 创新点 B：端到端梯度引导的策略搜索 (Gradient-Guided Policy Search)
*   **传统做法**：RL 通过随机噪声（高斯探索）来试错，效率极低。
*   **本项目**：利用 MPX 提供的解析梯度。机器人不需要随机尝试“迈左腿还是迈右腿”，物理模型的梯度会直接告诉它：“迈左腿会摔倒，因为左边摩擦力不够”，从而直接更新策略。
*   **价值**：将训练所需的样本量从“亿级”降低到“百万级”甚至更低。

### 创新点 C：自适应 MPC 参数学习 (Context-Aware MPC Tuning)
*   **实现**：视觉网络不仅输出地形，还实时输出 MPC 的超参数（如 $Q, R$ 权重矩阵、预测步长 $N$、摩擦系数 $\mu$）。
*   **场景**：看到前方是冰面，网络自动调低 MPC 内部的摩擦系数约束；看到前方是乱石，自动调大阻尼项权重。
5.  **RL-Augmented MPC (2024-2025 最火的竞品)**
    *   **论文**: *RL-augmented Adaptive MPC for Bipedal Locomotion over Challenging Terrain* (ArXiv 2025)
    *   **局限性**：他们是用 RL 调整 MPC 的参数（比如改摩擦系数），而不是改 Cost Function 的地形。
    *   **优势**：直接改地形 Cost，比调参数更直观，避障能力更强。且基于MPC的物理函数损失能一路传到视觉网络，实现端到端训练。
6.  **Visual Whole-Body Control (分层架构)**
    *   **论文**: *Visual Whole-Body Control for Legged Loco-Manipulation* (CoRL 2024)
    *   **链接**: [ArXiv](https://arxiv.org/abs/2403.16967)
    *   **局限性**：视觉层（High-level）和控制层（Low-level）是分开训练的，没有端到端求导。
    *   **优势**：梯度是从力矩直接传回像素的 (Pixel-to-Torque gradients)，是真正的 End-to-End。
7.  **LocoMamba / Transformer-based RL (纯 RL)**
    *   **论文**: *Learning Vision-Guided Quadrupedal Locomotion End-to-End with Cross-Modal Transformers* (ICLR/CoRL)
    *   **局限性**：纯黑盒，不知道网络学到了什么。
    *   **优势**：中间层是 MPC，可以把生成的 Cost Map 画出来：“网络把这块草地当成了平地，把那个水坑当成了高墙”。可解释性完胜。

## 3. 可行性分析 (Feasibility Analysis)

### 3.1 理论可行性
*   **数学基础**：隐函数定理（Implicit Function Theorem）允许我们对优化问题的解（MPC 的输出）求导，而不需要展开所有的迭代步骤。这在理论上保证了反向传播的稳定性。
*   **物理模型**：全动力学模型（Full Dynamics）虽然复杂，但在 JAX 中可以通过自动微分（AD）处理。

### 3.2 工程可行性
*   **算力支撑**：NVIDIA Jetson Orin 或 RTX 4090 已具备运行 JAX 并行计算的能力。
*   **软件生态**：
    *   **求解器**：基于开源的 `MPX` 或 `MpcJax`。
    *   **仿真器**：使用 `MJX` (MuJoCo XLA) 实现全 GPU 仿真，支持数千个机器人并行训练。
    *   **视觉骨干**：使用轻量级的 ViT 或 ResNet。

### 3.3 潜在风险与对策
*   **风险 1：梯度爆炸/消失**。
    *   *对策*：使用 Implicit Differentiation（隐式微分）代替直接展开迭代（Unrolling）；使用正则化项约束 Cost 范围。
*   **风险 2：Sim-to-Real 鸿沟**。
    *   *对策*：在仿真中引入大量域随机化（Domain Randomization）。
*   **风险 3：对比模块demo不容易在本地跑起来**。
    *   *对策*：可以寻求其他有类似积累的实验室课题组合作完成相关内容

## 5. 代价与成本 (Costs & Requirements)

*   **训练阶段**：需要高性能 GPU 服务器（如 A100/H100 集群）进行大规模并行仿真训练。相比纯 RL，单步计算量大（因为要解 MPC），但总步数少。
*   **推理阶段**：机载电脑（如 Orin NX）需支持 CUDA 加速。MPC 求解频率需达到 50Hz 以上，这对代码优化（JIT Compilation）要求极高。建议   纯仿真，真机成本过高

## 6. 评价方法与量化指标 (Evaluation Metrics)

为了验证创新点的有效性,设计以下实验和指标：

### 6.1 仿真环境 (Simulation Metrics)
*   **地形穿越成功率 (Success Rate)**：
    *   测试集：随机生成的碎石路、台阶（5-15cm）、湿滑地面（摩擦系数 0.1-0.5）。
    *   *目标：相比 Baseline (Perceptive MPC)，成功率提升 20% 以上。*
*   **样本效率 (Sample Efficiency)**：
    *   达到相同奖励值（Reward）所需的训练步数。
    *   *目标：比纯 RL (PPO) 快 10 倍以上。*
*   **能量效率 (Cost of Transport, COT)**：
    *   $COT = \frac{E}{mgd}$。验证由 MPC 产生的动作是否比 RL 更节能、更平滑。

**2. 实验场景 (Scenarios in MuJoCo Playground):**
*   **Stepping Stones (梅花桩)**: 考验落脚点精度（这是 MPC 的强项，RL 的弱项）。
*   **Slippery Patch (隐形冰面)**: 考验在线适应能力（梯度回传能快速修正摩擦力估计）。
*   **Moving Obstacle (动态避障)**: 考验实时重规划能力（MPX 强项）。

### 6.2 真机实验 (Real-World Metrics)理论上，真机其实未必需要有
*   **位置跟踪误差 (Tracking Error)**：RMSE (Root Mean Square Error)。
*   **抗扰动能力 (Disturbance Rejection)**：
    *   用 5kg 力侧向推机器人，记录恢复平衡所需的时间和步数。
*   **计算延迟 (Latency)**：
    *   从图像输入到力矩输出的总耗时（ms）。

## 7. 总结 (Conclusion)

本项目并没有试图推翻现有的控制理论，而是利用最新的 **Differentiable Programming (可微编程)** 技术，填补了“视觉感知”与“最优控制”之间的最后一道鸿沟。

通过将 MPC 作为一个**可学习、可求导的层**嵌入到神经网络中，我们既保留了基于模型控制的物理可解释性和安全性，又获得了深度学习对非结构化环境的强大适应能力。
